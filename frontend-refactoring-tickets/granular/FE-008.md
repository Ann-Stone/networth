## Notes

- The store does not expose mutation helpers (create/update/delete). Views call the API functions directly and then call `fetchX()` to refresh state. This keeps stores simple.
- Do not import any other store inside this store.
- `fetchBudgets` takes `year: number` param; it stores the result in `budgets` for the currently selected year. The view manages which year is selected.
