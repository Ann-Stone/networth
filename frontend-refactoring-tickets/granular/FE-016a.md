## Notes

- **Scope discipline**: this ticket only adds primitives + sidebar refactor. Do NOT preview / pre-implement Phase 3 views. If a primitive can't be designed without seeing a real consumer, defer it (YAGNI).
- **Element Plus integration**:
  - Tabs / table-expand / number inputs / pagination — use Element Plus directly in views; do not wrap them here. (`el-tabs`, `el-table`'s `expand` type, `el-input-number`, `el-pagination`.)
  - `FormDialog` is the one wrapper we need because every CRUD ticket repeats the same dialog/form/footer plumbing.
- **`MoneyDisplay` extension** is additive — `size` defaults to `'md'` so existing usages stay correct. Existing `positive` prop semantics unchanged.
- **Tone tokens**: `primary` = `--el-color-primary` (sage `#8fa79b`), `rose` = `--accent-rose` (`#b58d8d`). No new colors invented; everything routes through CSS vars defined in FE-013.
- **Dark mode parity**: every new component must render correctly in both light (default) and `html.dark` mode. Use the dual class pattern (`bg-white dark:bg-surface-dark`) consistently.
- **No store imports**: every new component is stateless and prop-driven; only `useConfirm` is allowed to talk to Element Plus's imperative API.
- **Sidebar polish closure**: sub-tasks 9-10 close the gap flagged at the end of Phase 2 (group toggle button styling). After this ticket, no follow-up sidebar polish should remain.
