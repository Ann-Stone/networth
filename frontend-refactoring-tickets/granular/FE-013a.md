# FE-013a — Default to template.html dark theme + dark/light toggle

> Depends on: FE-013 (palette CSS vars already in main.css)
>
> Revises FE-013 Note 20 ("Dark mode is opt-in; light mode is default"):
> the user-supplied design reference `template.html` ships with
> `<html class="dark">` as its baseline, and Phase 3 verification
> (`/networth/dashboard`) confirmed every component already declares
> `dark:` Tailwind variants. The visual mismatch is purely the missing
> default class. This ticket flips the default to dark and adds a
> minimal user-controllable toggle so light mode remains reachable.

## Notes

- **Do not** invent new color tokens. Reuse the palette declared in
  `view/src/assets/main.css` by FE-013 (`background-dark`, `surface-dark`,
  `cream`, `muted-text`, `primary`, `accent-rose`).
- **Do not** modify any `api/`, `account-book-API/`, or
  `account-book-view/` file.
- The toggle stays per-browser (localStorage); no backend persistence —
  there is no auth layer per the project decision log.
- `template.html` is the visual reference. The toggle button itself does
  not appear in `template.html`; place it where the locale switch
  currently lives (top-right of `Navbar`) so the page header stays
  uncluttered.
