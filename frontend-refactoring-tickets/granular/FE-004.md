## Notes

- `GET /dashboard/alarms` is the dashboard-version alarm query (returns upcoming alarms); it differs from `GET /settings/alarms` (management CRUD). Both exist in the API.
- Read `api/docs/api-reference.md` Dashboard section for exact response shapes of `DashboardSummary`, `DashboardBudget`, `DashboardGift`.
- Selection endpoints return lightweight objects for populating `<el-select>` dropdowns; they do not contain full resource records. Check the `/utilities/selections/*` section in api-reference.md for exact shapes.
- The import endpoints use `BackgroundTasks` server-side; the API returns immediately with a `202`-style success. Return value is `Promise<unknown>`.
