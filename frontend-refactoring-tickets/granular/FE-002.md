## Notes

- **All field names are snake_case** — this matches the backend API. Do not camelCase them; the frontend does not transform field names.
- Read `api/docs/api-reference.md` carefully for each domain's response shapes before writing types. When ambiguous, use `api/docs/openapi.json` for exact schemas.
- Y/N boolean flags (`in_use`, `is_calculate`, `has_closed`) are `string` type in TypeScript, not `boolean`.
- Nullable optional fields use `T | null` with `?` (e.g., `memo?: string | null`).
- Every interface must be `export`ed.
- Do not add types for items not yet in the API spec. If you need a UI-only type (e.g., for a form field), prefix it clearly (e.g., `FormState`).
